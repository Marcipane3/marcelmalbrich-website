# Phase 1 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Scope note:** This phase's only UI surface is a single placeholder page proving the deploy pipeline works end-to-end (CONTEXT.md decision D-07). It is deliberately minimal — no visual identity, no component system, no design tokens. The real design system (colors, typography, centralized Tailwind tokens, responsive shell) is Phase 2's scope and is intentionally NOT specified here, to avoid work Phase 2 would just replace. Anything below marked "Phase 2 owns" must not be treated as a finalized decision — it exists only so this phase's placeholder doesn't accidentally look broken or unreadable.

## Structural / Accessibility Requirements (this phase's actual bar)

Since there is no visual design to validate against in the traditional sense, the checker should instead validate these structural minimums:

- [ ] Page uses semantic HTML (e.g. a single `<h1>` for the name, a `<p>` for the tagline — not divs styled to look like headings)
- [ ] Page has a `<title>` tag (e.g. "Marcel Malbrich") set via Next.js metadata
- [ ] Body text passes WCAG AA contrast against the background (4.5:1 minimum)
- [ ] Page is not visually broken on a mobile-width viewport (375px) — text wraps, no horizontal scroll, no overflow
- [ ] No dark mode / theme toggle present (matches DSGN-01 project-wide constraint, enforced early even though Phase 2 owns the full theme)
- [ ] No placeholder/lorem ipsum filler beyond the name + tagline — content is real, just minimal
