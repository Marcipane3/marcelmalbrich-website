## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking issue] Removed scaffold-generated `CLAUDE.md` and `AGENTS.md` from `main/`**
- **Found during:** Task 1, immediately after scaffold completion
- **Issue:** Next.js 16's `create-next-app` now auto-generates `main/CLAUDE.md` (a one-line `@AGENTS.md` reference) and `main/AGENTS.md` (Next.js 16 breaking-change agent-rules notice) by default. A `CLAUDE.md` inside `main/` would shadow or confuse precedence with the project's actual root-level `CLAUDE.md` conventions file in future sessions — not part of the plan's file list, not requested, and actively conflicting with the project's existing documentation structure.
- **Fix:** Deleted both files from `main/` before committing.
- **Files modified:** `main/CLAUDE.md` (deleted), `main/AGENTS.md` (deleted) — neither was ever committed.
- **Commit:** Included in `aa1a6a8` (deleted prior to first add/commit, so they never entered history).

**2. [Rule 3 - Blocking issue] Switched git remote protocol from SSH to HTTPS**
- **Found during:** Task 2, first `git push -u origin master` attempt
- **Issue:** `gh repo create --source=. --remote=origin` configured the new `origin` remote using the SSH URL (`git@github.com:...`), per `gh`'s default git protocol setting on this machine. The push failed with "Host key verification failed" — the local SSH client has no trusted host key for `github.com` yet.
- **Fix:** Ran `gh auth setup-git` and switched the remote URL to HTTPS (`https://github.com/Marcipane3/marcelmalbrich-website.git`) via `git remote set-url origin`, relying on `gh`'s existing token-based git credential helper instead of debugging/accepting SSH host keys.
- **Files modified:** none (git config only, not a tracked file).
- **Commit:** N/A (no file change — remote config only).

No other deviations. Plan executed as written otherwise.
